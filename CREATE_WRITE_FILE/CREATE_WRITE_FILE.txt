
#------------Creating and Writing the contents in to the File-------------#

 1) touch : creates a file if it doesnâ€™t exist, or updates if it does.

ubuntu@ip-172-31-8-112:~$ cd DevOps/
ubuntu@ip-172-31-8-112:~/DevOps$ touch hello.txt  <--------------- Using touch command here to Create a file.
ubuntu@ip-172-31-8-112:~/DevOps$ ls
hello  hello.txt                          <---------- Here we can see "hello.txt" a text file got created used by touch command.

 2) > : prints text and saves it into a file , If the file doesn't exist then it will create a new one.

ubuntu@ip-172-31-8-112:~/DevOps$ echo "hello doston" > hello.txt  <-------------- Displaying "hello doston" in screen as well as writing in a file name as "hello.txt".
ubuntu@ip-172-31-8-112:~/DevOps$ cat hello.txt                   <------------------ Here through cat command we can see the content to ensure, it wrote the content really inside of the file!!! or only displaying in the screen.

hello doston                            <--------- Here we can see the same content "hello doston".

ubuntu@ip-172-31-8-112:~/DevOps$ echo "hello sir" > hello.txt  <--------------- "hello.txt" already exist so it will Overwrite this text "hello sir" in "hello.txt" Otherwise it would have created a new file in the name of "hello.txt".

ubuntu@ip-172-31-8-112:~/DevOps$ cat hello.txt
hello sir

ubuntu@ip-172-31-8-112:~/DevOps$ echo "Kyun re" > me.txt  <---------------- "me.txt" didn't exist so it created the file name as "me.txt" as well as displayed the content "Kyun re" in screen also it wrote inside of the file "me.txt".
ubuntu@ip-172-31-8-112:~/DevOps$ ls
hello  hello.txt  me.txt                        <----------- "me.txt" a text file got created where is written something which is  "Kyun re".
ubuntu@ip-172-31-8-112:~/DevOps$ cat me.txt
Kyun re                                          <------------- Here we can see the same content "Kyun re".

 3) >> : It appends the text hello doston to the file hello.txt , If the file does not exist, it creates it.

ubuntu@ip-172-31-8-112:~/DevOps$ echo "hello doston" >> hello.txt         <-------------- In this way we can append the contents to the "hello.txt" file. Just how we did "hello doston" after "hello sir".

ubuntu@ip-172-31-8-112:~/DevOps$ cat hello.txt 
hello sir
hello doston                                  <--------------- Here "hello doston" got append after the "hello sir" but in new line.
